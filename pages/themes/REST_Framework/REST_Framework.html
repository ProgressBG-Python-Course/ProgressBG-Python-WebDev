Web Services

Web services are the software component that supports machine to machine interaction over network.
The related data is stored on a remote server and transmitted to the client machine through APIs that provide web services for third-party users.
APIs can use different architectures to transfer data from the server to the client.

REST vs SOAP

SOAP REST
An XML based message protocol An architectural style protocol
Uses WSDL for communication between client and service Can use JSON, XML, plain text to send and receive data
Invokes services by calling RPC methods Calls services via URL path (API endpoint)
Does not return human-readable format Returns plain ASCII: JSON, XML,...
Relies on HTTP and SMTP/FTP... Relies only on HTTP
Difficult to process the output with JavaScript JavaScript can easily parse JSON
Bulky, bigger learning curve Lightweight, better performance, less CPU intensive

Serializers

Serializers allow complex data such as querysets and model instances to be converted to native Python datatypes that can then be easily rendered into JSON, XML or other content types
The serializers in REST framework work very similarly to Django's Form and ModelForm classes

Declaring Serializers

Let's create the Python object, which we'll serialize: 

			class Comment(object):
				def __init__(self, email, content):
					self.email = email
					self.content = content

			comment = Comment(email='ada@byron.com', content='some comment')
Now, we can declare the CommentSerializer:

			from rest_framework import serializers

			class CommentSerializer(serializers.Serializer):
				email = serializers.EmailField()
				content = serializers.CharField(max_length=200)

ViewSets

With ViewSets we can combine the logic for a set of related views in a single class, called a ViewSet
A ViewSet class is simply a type of class-based View, that provides actions such as list(), create() an so on.
The method handlers for a ViewSet are only bound to the corresponding actions at the point of finalizing the view, using the .as_view() method.
Typically, rather than explicitly registering the views in a viewset in the urlconf, you'll register the viewset with a router class, that automatically determines the urlconf for you.

REST framework adds support for automatic URL routing to Django, and provides you with a simple, quick and consistent way of wiring your view logic to a set of URLs.
Reference: routers

Set the urls path for the todo_rest_api:

			path('todos_rest/', include('todo_rest_api.urls')),

Create ModelSerializer

We need ModelSerializer to convert model instances to JSON which will be easily consumed by the front-end
ModelSerializer class

  		from rest_framework import serializers
  		# reuse the todo_app.model:
  		from todo_app.models import Task

  		class TaskSerializer(serializers.ModelSerializer):
  		  class Meta:
  		    model = Task
  		    fields = '__all__'

Create the ViewSet

The viewsets base class provides the implementation for CRUD operations by default
The actions provided by the ModelViewSet class are list(), retrieve(), create(), update(), partial_update(), and destroy().
ModelViewSet class

  		from django.shortcuts import render
  		from rest_framework import viewsets
  		from .serializers import TaskSerializer
  		from todo_app.models import Task

  		class TaskView(viewsets.ModelViewSet):
  		  serializer_class = TaskSerializer
  		  queryset = Task.objects.all()

Create the Router


  		from django.urls import path, include
  		from rest_framework import routers
  		from . import views

  		router = routers.DefaultRouter()
  		router.register(r'todos', views.TaskView, 'todo')

  		urlpatterns = [
  		  path('api/', include(router.urls))
  		]
